O que foi aprendido:

  • git --version - para ver qual é a versão do git instalada;
  • git init - para inicializar um novo repositório local;
  • git status - para ver um status atualizado sobre o repositório, se existem pendências ou se está tudo ok;
  • git add \index.html - após a criação de um arquivo, usar este comando para adicioná-lo ao fluxo do git;
  • git add . - para adicionar todas as alterações existentes ao fluxo de versionamento do git;
  • git commit -a -m "create index.html" - para fazer o commit das alterações;
  • git remote add origin https://github.com/juniortex/tutorial-git.git - para adicionar uma origem remota ao repositório, ou seja, fazer a conexão entre a pasta do repositório local com o repositório do github;
  • git branch -M main - para nomear o branch principal como main e parear a branch principal local com a remota, sincronizando-as.
  • git push -u origin main - para enviar o conteúdo do repositório local para o remoto, atualizando-o para que ambos tenham o mesmo conteúdo.
  • git checkout -b "teste" - cria uma branch nova com o nome 'teste' e muda o nosso fluxo para ela.
  • git push --set-upstream origin teste - como a branch "teste foi criada localmente, através do terminal, ela ainda não existe no repositório remoto, logo, se faz necessário o uso deste comando."
  • git push - enviar as alterações realizadas na branch local para o repositório remoto, quando a branch já existe em ambos."
  • git checkout main - para trocar de branch, alterando assim, o fluxo de versionamento.
  • git branch - ver a lista de branch's existentes na aplicação e confirmar em qual branch estamos trabalhando.
  • git merge teste - para fazer uma junção de conteúdos, pegando o conteúdo da branch especificada (teste) e passando para a branch atual (main). Esse processo deixará as duas branch's iguais de forma local, ainda faltará fazer push para o remoto.
